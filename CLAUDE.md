## Project Structure

```

_includes/               # Reusable components
_pages/                  # Site pages (about, contact, blog)

assets/{css,js,images}/  # Static assets
index.md, 404.md         # Must be in root
```

## Critical Rules

### URL Structure

**Permalink format:** `/blog/YYYY/post-title/` (e.g., `/blog/2024/DNS-Guardrails-with-dnscrypt-proxy/`)

**New posts:** Place in `_posts/YYYY/YYYY-MM-DD-Title.md` - no redirect_from needed (new URL)

**Existing posts:** Already have `redirect_from: /Old-Title/` to maintain backward compatibility

**Redirects:** Jekyll-redirect-from plugin handles old URLs → new URLs
- Production: Absolute URLs (https://www.ecliptik.com/blog/...)
- Development: Relative URLs (/blog/...) via custom _layouts/redirect.html

### Tags (VERY IMPORTANT)

**ALWAYS** check `_scripts/README.md` before tag work.

- Lowercase only: `kubernetes` not `Kubernetes`
- Check existing: `ls tag/`
- Use canonical names: `macos`, `512ke`, `networking`, `terminal`, `smolweb`
- After changes: `python3 _scripts/tag_generator.py` from repo root
- Should have ~78 tags total

### Files to Never Modify

- ❌ `_posts/*.html` (2004-2011) - Historical HTML, never convert to Markdown
- ❌ `_gemini/`, `_gopher/` - Auto-generated by updatesmallweb.sh
- ❌ CNAME - Cloudflare handles DNS
- ❌ Reference `console.css` in layouts - use `console.min.css`

## Development

### Docker (Recommended)

```bash
docker compose up -d              # Start (localhost:4000 or jezebel:4000)
docker compose logs -f jekyll     # View logs
docker compose down               # Stop
docker compose build --no-cache   # Rebuild after Gemfile changes
```

**Note:** Docker uses both `_config.yml` and `_config_dev.yml` for relative redirects (works with any hostname)

### Manual Jekyll

```bash
bundle install
bundle exec jekyll serve --watch  # localhost:4000
# For development with relative redirects:
bundle exec jekyll serve --config _config.yml,_config_dev.yml --watch
```

### Theme & CSS

Edit `assets/css/console.css` then regenerate minified version:

```python
python3 << 'EOF'
import re
with open('assets/css/console.css', 'r') as f:
    css = f.read()
css = re.sub(r'/\*.*?\*/', '', css, flags=re.DOTALL)
css = re.sub(r'\s+', ' ', css)
css = re.sub(r'\s*{\s*', '{', css)
css = re.sub(r'\s*}\s*', '}', css)
css = re.sub(r'\s*:\s*', ':', css)
css = re.sub(r'\s*;\s*', ';', css)
css = re.sub(r'\s*,\s*', ',', css)
css = re.sub(r';\s*}', '}', css)
css = re.sub(r'}', '}\n', css)
with open('assets/css/console.min.css', 'w') as f:
    f.write(css.strip())
EOF
```

## Common Tasks

### New Post

```bash
_scripts/newpost.sh "Post Title"  # Creates _posts/YYYY/YYYY-MM-DD-Title.md
```

Frontmatter:
```yaml
---
layout: post
title: Your Post Title
category: linux
tags: linux kubernetes docker terminal  # Use canonical tags!
---
```

### Modify Tags

1. Edit post frontmatter
2. Verify canonical tags (`_scripts/README.md`)
3. `python3 _scripts/tag_generator.py`
4. Commit post + regenerated tag files

### Add Images

Locations: `assets/images/posts/`, `assets/images/`, `assets/images/{category}/`

```markdown
![Alt text](/assets/images/posts/image.jpg)
```

### Update Small Web

```bash
_scripts/updatesmallweb.sh -t all
```

## Troubleshooting

**Site won't build:**
```bash
docker compose logs jekyll
docker compose down && docker compose build --no-cache && docker compose up -d
```

**Tag pages missing:**
```bash
python3 _scripts/tag_generator.py
ls tag/*.md | wc -l  # Should be ~78
```

**Redirects not working in dev:** Ensure `_config_dev.yml` exists and Docker is using both config files

**Images not loading:** Check case-sensitive paths, verify file exists, check frontmatter
