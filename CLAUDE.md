## Project Structure

```

_layouts/, _includes/          # HTML templates and components
_pages/, _pages/years/         # Site pages and year archives
_posts/YYYY/                   # Blog posts by year (*.md and *.html from 2004+)

_gemini/, _gopher/             # Auto-generated (DO NOT MODIFY)
```

## Critical Rules

**URLs:** `/blog/YYYY/post-title/` format. New posts in `_posts/YYYY/YYYY-MM-DD-Title.md`. Existing posts have `redirect_from:` for backward compatibility.

**Tags (VERY IMPORTANT):**
- ALWAYS check `_scripts/README.md` first
- Lowercase only (`kubernetes` not `Kubernetes`)
- Use canonical names: `macos`, `512ke`, `networking`, `terminal`, `smolweb`
- After changes: `python3 _scripts/manage.py tags` (from repo root)
- Should have ~78 tags total

**Never Modify:**
- `_posts/*.html` (2004-2011 historical HTML)
- `_gemini/`, `_gopher/` (auto-generated)
- CNAME (Cloudflare handles DNS)

**CSS & JS:** Minified versions (`*.min.css`, `*.min.js`) generated via `manage.py minify` - layouts reference minified files for production

## Gopher Protocol Generation

**Generate for local testing (default):**
```bash
python3 _scripts/manage.py gopher  # localhost:7070
```

**Generate for production deployment:**
```bash
python3 _scripts/manage.py gopher --host gopher.club --port 70
```

**What it generates:**
- 30 markdown posts (2015-2026) → plaintext (70 columns)
- 63 tag pages with gophermaps
- 8 year archives (2011, 2015, 2017, 2021-2025)
- Static pages (about.txt, contact.txt)
- RFC1436-compliant gophermaps with correct host/port

**Output:** `_gopher/` directory with:
- `blog/YYYY/*.txt` - Posts by year
- `tags/*/gophermap` - Tag directories
- `gophermap` - Root navigation

**Configuration:**
- **Default (testing):** localhost:7070
- **Production:** gopher://gopher.club:70/1/users/ecliptik/

**Options:**
```bash
python3 _scripts/manage.py gopher --force              # Regenerate all
python3 _scripts/manage.py gopher --columns 80         # Custom width
python3 _scripts/manage.py gopher --host gopher.club   # Production host
python3 _scripts/manage.py gopher --port 70            # Production port
```

**Local Testing:**

**Deploy to SDF.org:**

## Gemini Protocol Generation

**Generate for local testing (default):**
```bash
python3 _scripts/manage.py gemini  # localhost:1965
```

**Generate for production deployment:**
```bash
python3 _scripts/manage.py gemini --host sdf.org --port 1965 --base-url "gemini://sdf.org/ecliptik/"
```

**What it generates:**
- 30 markdown posts (2015-2026) → gemtext (.gmi files)
- 63 tag pages with gemtext links
- 8 year archives (2011, 2015, 2017, 2021-2025)
- Static pages (about.gmi, contact.gmi)
- Native gemtext format with proper syntax

**Output:** `_gemini/` directory with:
- `blog/YYYY/*.gmi` - Posts by year
- `tags/*.gmi` - Tag pages
- `index.gmi` - Root navigation

**Configuration:**
- **Default (testing):** localhost:1965
- **Production:** gemini://sdf.org/ecliptik/

**Options:**
```bash
python3 _scripts/manage.py gemini --force              # Regenerate all
python3 _scripts/manage.py gemini --host sdf.org       # Production host
python3 _scripts/manage.py gemini --port 1965          # Production port
```

**Local Testing:**
```bash
# Generate (localhost:1965)
python3 _scripts/manage.py gemini

# Verify
bash tests/verify_gemini.sh

# Generate TLS certificates (first time only)
./smallweb/gemini/config/certs/generate-certs.sh

# Start test server
docker compose -f smallweb/gemini/docker-compose.yml up -d

# Test
amfora gemini://localhost:1965/
# Or from Tailscale network:
amfora gemini://jezebel:1965/

# Stop
docker compose -f smallweb/gemini/docker-compose.yml down
```

**Deploy to SDF.org:**
```bash
# Generate for production
python3 _scripts/manage.py gemini --host sdf.org --port 1965

# Upload to SDF
rsync -avz --delete _gemini/ username@sdf.org:gemini/

# Set permissions
ssh username@sdf.org 'chmod -R a+rX ~/gemini/'
```

**Key differences from Gopher:**
- Native gemtext format (not plaintext)
- NO line wrapping (client handles it)
- Links: `=> url text` (not tab-delimited selectors)
- Headers: `#` `##` `###` (native support)
- TLS required (self-signed cert for Docker testing)
- Port 1965 (not 70)
- .gmi extension (not .txt)

**Page structure:**
- **Site map:** All pages include site map navigation at the top (Home, About, Contact, Blog, Tags)
  - Blog posts: Site map appears after ASCII art header, before title
  - Archive/index pages: Site map appears after breadcrumbs, before title
  - Static pages: Site map appears at the very top
- **Blog posts:** ASCII art header → Site map → Title → Date/Tags → Content → Links section
- **Navigation:** Breadcrumbs on archive/index pages (← Home, ← Blog, etc.)

**Link and image handling:**
- **Text links:** Footnote style with inline text and `[N]` references
  - Example: `I discussed the origins[1] and enhancements[2] here.`
  - Links section at bottom: `=> /blog/2025/Post.gmi [1] origins`
- **Images:** Standalone gemtext links on their own line
  - Example: `=> /assets/images/posts/img.jpg Image description`
- **Internal blog links:** Automatically converted to gemini paths
  - `https://www.ecliptik.com/Post/` → `/blog/YYYY/Post.gmi`
- **Assets:** Automatically copied to `_gemini/assets/` during generation
- **Docker testing:** Configured for Tailscale hostname `jezebel`

See `smallweb/gemini/DEPLOY.md`, `_scripts/README.md`, and `tests/README.md` for detailed documentation.
